## Key Learnings

1. Fix the entry price bug first -- performance.json logging entry=0 for all trades makes the whole dataset unreliable.
2. Verify the EOD closer is running -- MSFT at closed=false at market close is a red flag. Check cron logs.
3. Three losers in a row = time to audit. Not necessarily system failure, but need to check if signal direction is calibrated to current macro (broad tech selloff, tariff uncertainty).
4. Add a same-ticker cooldown after consecutive losses. AAPL fired twice in a row and lost both -- that should not happen without a reset condition.
5. The MSFT put thesis itself was sound -- massive institutional put flow, confirming gap, clean earnings window. If it loses, the model was still right to fire. Process vs outcome.
